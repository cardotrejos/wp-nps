# FlowPulse - Project Documentation Index

> WhatsApp-native customer feedback platform leveraging Kapso WhatsApp Flows for NPS, CSAT, CES surveys.

## Project Overview

| Attribute    | Value                           |
| ------------ | ------------------------------- |
| **Type**     | Turbo Monorepo (6 parts)        |
| **Stack**    | Better-T Stack                  |
| **Language** | TypeScript (strict)             |
| **Frontend** | React 19 + Vite + TailwindCSS 4 |
| **Backend**  | Bun + Elysia + oRPC             |
| **Database** | PostgreSQL + Drizzle ORM        |
| **Auth**     | Better Auth (email/password)    |

## Quick Reference

### Parts

| Part       | Type         | Path              | Tech Stack                                    |
| ---------- | ------------ | ----------------- | --------------------------------------------- |
| **web**    | Web Frontend | `apps/web`        | React 19, Vite, TanStack Router/Query, shadcn |
| **server** | Backend API  | `apps/server`     | Bun, Elysia, oRPC Server, Better Auth         |
| **api**    | Library      | `packages/api`    | oRPC, Zod, OpenAPI                            |
| **db**     | Library      | `packages/db`     | Drizzle ORM, PostgreSQL                       |
| **auth**   | Library      | `packages/auth`   | Better Auth config                            |
| **config** | Library      | `packages/config` | Shared TypeScript config                      |

### Entry Points

| App    | Entry                      | URL                   |
| ------ | -------------------------- | --------------------- |
| Web    | `apps/web/src/main.tsx`    | http://localhost:5173 |
| Server | `apps/server/src/index.ts` | http://localhost:3000 |

### Key Commands

```bash
pnpm dev          # Start all apps
pnpm build        # Build all packages
pnpm check-types  # Type checking
pnpm db:push      # Push schema to database
pnpm db:studio    # Open Drizzle Studio
```

### For New Developers

```bash
# 1. Clone and install
git clone <repo-url>
cd wp-nps
pnpm install

# 2. Configure environment
cp .env.example .env
# Edit .env with your DATABASE_URL

# 3. Setup database
pnpm db:push

# 4. Start development
pnpm dev

# 5. Open in browser
# Frontend: http://localhost:5173
# Backend: http://localhost:3000
```
